# X 中文垃圾号识别 / 一键屏蔽

一个轻量的浏览器用户脚本（Tampermonkey），自动识别并屏蔽 X（Twitter）上泛滥的**中文色情 / 引流垃圾号**——就是那些 `同城上门`、`寻固炮`、`点击主页`、`t.me/xxx 加我` 之类的回复机器人。

## 它和老式"屏蔽名单"插件有什么不同

老插件收集的是一份**已知坏账号名单**（按身份屏蔽）。但这些号是一次性的、每天批量新生成，名单永远追不上 → 打地鼠必输。

本工具**不维护名单**，而是像邮箱的垃圾邮件过滤器一样，对**当面出现的每个账号实时按特征判断**：

- 用户名是不是「英文名 + 一长串随机数字」（如 `@Catheri47071247`）
- 显示名 / 正文里有没有 `同城上门`、`寻固炮`、`点击主页`、`日泡平台`、`秒约` 等关键词
- 有没有 `t.me/` 电报链或 `xxx.top` 这类短引流域名
- 有没有 🌸✈️💕 这类装饰 emoji 堆砌

命中特征到一定分数才出手 → 换 100 个新号也照样识别，规则只写一次。

> ⚠️ 设计上**单看"名字+数字"不会屏蔽**，避免误伤那些真人 @ 机器人的正常账号。

## 不是 AI agent，不卡

运行时**没有 AI、没有独立进程**。它就是几 KB 的 JavaScript，由浏览器当作网页的一部分来跑，和广告拦截器一个原理，开销可忽略。AI 只用在"前期把特征提炼成规则"，不在日常回路里。

## 安装（约 30 秒）

1. 给浏览器装一次 [Tampermonkey](https://www.tampermonkey.net/)（Chrome / Edge / Firefox / Safari 均可）。
2. 打开 `x-spam-blocker.user.js`，Tampermonkey 会弹出安装框 → 点「安装」。
3. 打开 x.com 正常刷即可。命中的垃圾号会被标红，并出现「屏蔽」按钮。

## 两种模式

打开脚本顶部，改 `MODE`：

- `'mark'`（默认，最安全）：只标红 + 给「屏蔽」按钮，你点了才屏蔽。建议先用这个观察准不准。
- `'auto'`：自动屏蔽命中的号（带随机间隔 + 单次会话上限，防风控）。看顺眼了再开。

关键词列表都在脚本顶部，中文注释，可自行增删。

## 本地验证 / 维护

本 repo 带一个零依赖回归 harness，用来防止规则和 X DOM 适配退化：

```bash
npm test
```

它会直接读取当前 `x-spam-blocker.user.js`，验证：

- 测试页加载当前脚本，不嵌入过期副本。
- 启动日志版本号和 userscript metadata 一致。
- 新波无敏感词中文垃圾号会被标记，正常短回复不会被误伤。
- 同一垃圾号出现多次时，每个可见卡片都会标记；自动模式只去重屏蔽请求。
- X 先渲染 handle、后补全文本时，会按内容签名重新判断。
- `/settings/blocked*` 页面会注入已屏蔽账号 JSON 采集器。

手工看样式时，可直接打开 `test/fixture.html`；它引用当前主脚本，避免 fixture 和发布脚本分叉。

## 适用范围

- ✅ 桌面浏览器（Chrome / Edge / Firefox / Safari / Brave / Arc 等）
- ⚠️ 手机浏览器一般不支持扩展/脚本（iOS 需配合 Userscripts 等 App，安卓需 Firefox/Kiwi）。本工具定位为**桌面端**。

## 状态

早期版本。X 的页面结构和内部接口偶有变动，可能需要微调选择器。请先用默认 `mark` 模式验证准确率，再考虑开 `auto`。

当前加固点：

- 事件驱动扫描，不做闲置轮询。
- 按卡片内容短签名复扫，适配 X SPA 异步渲染。
- 重复 handle 的可见卡片都会标记；自动屏蔽请求按 handle 去重并限速。
- 重复卡片联动置灰只保留最近窗口的 DOM 引用，避免长时间浏览内存膨胀。
- 已屏蔽账号页可导出 JSON 正样本，用于后续校准中文批量垃圾号特征。
